# VCS Frontend - Git DAG & Snapshot Data Model

## Architecture Overview

The VCS frontend now uses a **Directed Acyclic Graph (DAG)** and **snapshot-based commit model** that simulates real Git internals, while maintaining all existing UI components unchanged.

---

## Core Data Model

### 1. **Blobs** (`src/data/git/blobs.js`)
Individual file contents stored separately. Multiple snapshots can reference the same blob if a file hasn't changed.

```javascript
{
  blobId: 'blob-001',
  content: ['file contents as array of lines'],
  size: 65,
  extension: 'js',
  language: 'javascript'
}
```

**Benefits:**
- Deduplication: Same file content across commits → same blob
- Memory efficient
- Mirrors Git's blob storage

---

### 2. **Trees** (`src/data/git/trees.js`)
Directory snapshots representing the complete repository state at a point in time.

```javascript
{
  treeId: 'tree-001',
  entries: [
    { name: 'src', type: 'tree', treeId: 'tree-001-src' },
    { name: 'README.md', type: 'blob', blobId: 'blob-005' }
  ]
}
```

**Features:**
- Nested tree structures for directories
- References to blobs for files
- Represents full repository snapshot
- Every commit points to exactly one tree

---

### 3. **Commits** (`src/data/git/commits.js`)
Form the DAG. Each commit has:

```javascript
{
  commitId: 'a1b2c3d',
  shortHash: 'a1b2c3d',
  message: 'Commit message',
  author: 'Developer Name',
  email: 'dev@example.com',
  timestamp: '2026-06-27 09:15',
  parentCommitIds: ['e4f5g6h'],  // DAG edges - one or more parents
  treeId: 'tree-001',             // Points to complete snapshot
  branch: 'main',
  tags: ['v1.0'],
  mergeCommit: false
}
```

**DAG Properties:**
- `parentCommitIds: []` → Root commit
- `parentCommitIds: ['abc123']` → Normal commit
- `parentCommitIds: ['abc123', 'xyz456']` → Merge commit (two parents)
- No circular references guaranteed

---

### 4. **Branches** (`src/data/git/branches.js`)
Simple pointers to commits:

```javascript
{
  main: {
    name: 'main',
    commitId: 'a1b2c3d'  // Points to latest commit
  },
  'feature/auth': {
    name: 'feature/auth',
    commitId: 'e4f5g6h'
  }
}
```

**Behavior:**
- Each branch → latest commit on that branch
- Switching branches updates HEAD
- Creating new commit updates branch pointer

---

### 5. **HEAD Pointer** (`src/data/git/head.js`)
Tracks current state per repository:

```javascript
{
  repositoryId: 'repo-1',
  currentBranch: 'main',
  currentCommitId: 'a1b2c3d',
  currentTreeId: 'tree-001'
}
```

**Updates on:**
- Branch checkout: `checkoutBranch(repoId, branchName)`
- New commit: Auto-updated by `createCommit()`

---

### 6. **Staging Area** (`src/data/git/staging.js`)
Files ready to commit:

```javascript
{
  repositoryId: 'repo-1',
  stagedFiles: [
    { name: 'README.md', status: 'modified', blobId: 'blob-005' }
  ],
  unstagedFiles: [
    { name: 'src/auth/session.js', status: 'modified', blobId: 'blob-001' }
  ]
}
```

---

### 7. **Working Directory** (`src/data/git/workingDirectory.js`)
Files being worked on (not yet staged):

```javascript
{
  repositoryId: 'repo-1',
  modifiedFiles: [...],
  addedFiles: [...],
  deletedFiles: [...]
}
```

**Separation:**
- Working Directory → Current edits
- Staging Area → Files ready to commit
- HEAD → Last committed state

## Utility Functions (`src/data/git/gitUtils.js`)

### Accessors
```javascript
getCommit(commitId)           // Get commit object
getTree(treeId)               // Get tree object
getBlob(blobId)               // Get blob object
```

### Traversal
```javascript
getParentCommits(commitId)    // Get parent commits (DAG traversal)
getCommitHistory(commitId)    // Walk DAG to get full history
getBranchCommits(repoId, branch)  // Get all commits on branch
```

### Diff Generation
```javascript
generateDiff(treeId1, treeId2)      // Compare two trees
generateCommitDiff(commitId1, commitId2)  // Compare commits
```

**Dynamic diff generation:**
- Diffs not stored directly
- Generated by comparing trees
- Supports existing DiffViewer UI without changes

### Branch Operations
```javascript
checkoutBranch(repoId, branchName)  // Update HEAD
getBranches(repoId)                 // Get all branches
```

### Tree Navigation
```javascript
getFileTree(treeId)  // Flatten tree into file list
```

### Merge Base (for conflict resolution)
```javascript
findMergeBase(commitId1, commitId2)  // Find common ancestor
```

## Data Flow: UI Components → Git Model

### Example: Viewing Commit Diff

1. **DiffViewPage** receives `commitId` from URL
2. Calls `getCommitFiles(commitId)` (adapter in `src/data/adapters.js`)
3. Adapter:
   - Gets commit object
   - Gets its parent commits
   - Calls `generateCommitDiff(parent, current)`
   - Which calls `generateDiff(parentTree, currentTree)`
   - Which compares blobs at each path
   - Returns diff with `oldContent` and `newContent`
4. **DiffViewer** component displays diffs (unchanged)

### Example: Switching Branches

1. **BranchManagerPage** calls `checkoutBranch(repoId, branchName)`
2. Updates HEAD pointer
3. RepoContext propagates change
4. **CommitHistoryPage** re-renders with new commits
5. **FileTree** shows new files
6. **DiffViewPage** shows new diffs

**Benefits:**

## Key Features

### Dynamic Diff Generation
- Diffs generated on-demand
- No storage of redundant data
- Supports full Git workflow

### DAG Commit Graph
- Normal commits: 1 parent
- Root commits: 0 parents
- Merge commits: 2+ parents
- History walking: BFS/DFS traversal

### Snapshot-Based Model
- Every commit has complete tree
- File deduplication via blobs
- No need to store file diffs

### Branch Pointers
- Branches are just refs to commits
- Switching updates HEAD
- Multiple branches can point to same commit

## Example: Complete User Flow

### 1. User opens app
```
Login → Select repo → CommitHistoryPage loads
```

### 2. CommitHistoryPage renders
```
RepoContext provides selectedRepo
selectedRepo.commits = getCommitHistory(HEAD.currentCommitId)
CommitList displays commits from DAG
```

### 3. User clicks commit
```
URL → /repo/repo-1/commit/a1b2c3d
DiffViewPage loads
getCommitFiles('a1b2c3d') generates diff
DiffViewer displays oldContent vs newContent
```

### 4. User switches branches
```
BranchManagerPage → checkoutBranch('repo-1', 'feature/auth')
HEAD.currentBranch = 'feature/auth'
HEAD.currentCommitId = 'e4f5g6h'
CommitHistoryPage re-renders with new history
```

## Summary

The VCS frontend now internally simulates Git's **DAG and snapshot architecture** while maintaining 100% compatibility with all existing UI components. This provides:

- **Realistic Git semantics** for future backend integration
- **Efficient storage** through blob deduplication
- **Dynamic diff generation** from tree snapshots
- **Flexible traversal** of commit history
- **Clean separation** between data model and UI
